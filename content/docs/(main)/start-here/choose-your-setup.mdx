## The short version

| If you want to... | Start with... | Why |
| --- | --- | --- |
| Learn SoulFire with the least setup | [GUI Mode](/docs/start-here/gui-mode) | Best first-run path and easiest to explore |
| Run long-lived sessions on a VPS or shared machine | [Dedicated Server](/docs/start-here/dedicated-server) | Keeps the backend running even when the GUI is closed |
| Automate from a terminal or run headless | [CLI Mode](/docs/start-here/cli-mode) | Good for scripts, SSH sessions, and keyboard-only workflows |
| Share one backend with multiple users or clients | [Dedicated Server](/docs/start-here/dedicated-server) | GUI and CLI clients can connect to the same remote server |

## What each part means

### GUI client

The GUI is the desktop app.
It is the best place to create instances, change settings, inspect state, and work with plugins or scripts.

### CLI client

The CLI is the terminal client.
It is useful when you want automation, SSH-based workflows, or a headless environment.

### Integrated server

The GUI and CLI both ship with an integrated SoulFire server.
This lets you run sessions locally on your own machine without deploying anything else.

### Dedicated server

A dedicated server is a separate SoulFire backend that runs on its own machine or as its own process.
GUI and CLI clients can connect to it remotely.

## Recommended paths

### New user

1. [Install SoulFire](/docs/start-here)
2. [Run your first session in GUI mode](/docs/start-here/gui-mode)
3. Use [How-to Guides](/docs/how-to) for accounts, proxies, plugins, and integrations

### Remote operator

1. [Deploy a dedicated server](/docs/start-here/dedicated-server)
2. Connect from the GUI with the server URL and access token
3. Use [How-to Guides](/docs/how-to) for ongoing administration tasks

### Automation-heavy user

1. Decide whether the CLI should run locally or against a remote server
2. Read [CLI Mode](/docs/start-here/cli-mode)
3. Use [Automation](/docs/automation) for the built-in operator automation surface
4. Keep [CLI Flags](/docs/reference/cli-flags) open as your reference
